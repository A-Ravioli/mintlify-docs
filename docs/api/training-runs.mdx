# Training Runs API

Create, monitor, and manage your training runs.

## List Runs

Get all your training runs.

<RequestExample>

```bash
curl https://api.frontier.com/v1/runs \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json
{
  "runs": [
    {
      "id": "run_abc123",
      "base_model": "meta-llama/Llama-3.2-1B",
      "status": "running",
      "created_at": "2025-01-15T10:00:00Z",
      "updated_at": "2025-01-15T10:30:00Z",
      "cost": 5.50
    }
  ]
}
```

</ResponseExample>

## Create Run

Start a new training run.

<RequestExample>

```bash
curl -X POST https://api.frontier.com/v1/runs \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "base_model": "meta-llama/Llama-3.2-1B",
    "training_data": "s3://your-bucket/training-data.jsonl",
    "config": {
      "learning_rate": 0.0001,
      "batch_size": 4,
      "num_epochs": 3
    },
    "integration_ids": ["integration_123"]
  }'
```

</RequestExample>

<ResponseExample>

```json
{
  "id": "run_abc123",
  "status": "queued",
  "estimated_cost": 5.50,
  "message": "Training run created successfully"
}
```

</ResponseExample>

## Cancel Run

Cancel a running training job.

<RequestExample>

```bash
curl -X POST https://api.frontier.com/v1/runs/run_abc123/cancel \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json
{
  "message": "Run cancelled successfully"
}
```

</ResponseExample>

## Request Body

### Create Run
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `base_model` | string | Yes | Base model identifier |
| `training_data` | string | Yes | URL to training dataset |
| `config` | object | No | Training hyperparameters |
| `integration_ids` | array | No | Integration IDs to use |

### Config Object
| Parameter | Type | Description |
|-----------|------|-------------|
| `learning_rate` | number | Learning rate (default: 0.0001) |
| `batch_size` | number | Batch size (default: 4) |
| `num_epochs` | number | Number of epochs (default: 3) |

## Run Status

Possible status values:
- `queued` - Waiting to start
- `running` - Currently training
- `completed` - Finished successfully
- `failed` - Encountered an error
- `cancelled` - Manually cancelled
